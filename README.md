# TechNova Solutions - Corporate Website Portfolio

A production-ready, fully responsive multi-page corporate website designed for **TechNova Solutions**, a software engineering and cloud consulting agency. This project is built using modern semantic HTML5, vanilla CSS3 variables, and object-oriented vanilla JavaScript.

---

## 🌟 Key Features

1. **Responsive Multi-Page Structure**:
   - Four detailed pages: `index.html` (Home), `about.html` (About), `services.html` (Services), and `contact.html` (Contact).
   
2. **Advanced Navigation**:
   - Sticky glassmorphic navbar that switches style on scroll.
   - Dynamic active page indicator matching the current URL.
   - Smooth slide-in mobile navigation menu drawer with scroll-locking overlay.

3. **Modern UX Components**:
   - **Scroll-Reveal Animations**: Intersection Observer triggers fade-in/slide-up effects on element view entry.
   - **Real-Time Counter**: Increments corporate metrics from 0 to the target metrics when scrolled into view.
   - **Interactive Form Validation**: Client-side name, email, phone number, and subject validation with inline error messaging and a success toast confirmation.
   - **Vector Asset Mockups**: Customized illustrations representing database arrays and cloud architecture models.

## 🎨 Design Guide & Palettes

- **Typography**: Poppins (Google Fonts)

- **Secondary Color**: `#1E293B` (Dark Slate)
- **Accent Color**: `#06B6D4` (Cyan Accent)
- **Background Color**: `#F8FAFC` (Slate Light Background)
- **Card/Component Background**: `#FFFFFF`
- **Text Color**: `#334155` (Slate Secondary Text)
- **Shadows**: Large soft shadows for premium depth styling.

## 🚀 How to Run and Test

1. Clone or download the project files into a local folder.
2. Locate `index.html` inside the project root.
3. Open `index.html` in any web browser (Chrome, Firefox, Safari, Edge, etc.).
4. Click on navigation links in the header or footer to browse between pages.
5. Resize the browser or test on a mobile device to inspect responsive layouts and mobile menus.
6. Navigate to `contact.html` to test input error validations and success toasts on submission.
